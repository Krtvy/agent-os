# Rootlabs metric glossary

One file per metric. Each file answers four questions:

1. **Plain-English definition** — what does this number mean to a human?
2. **Mechanical definition** — the exact SQL or pandas operation.
3. **Filters/exclusions** — what is deliberately left out (cancellations, promos, etc.).
4. **Provenance** — where the canonical query lives.

Read this before answering any metric question. If a metric is missing here,
note it in `\_audit/REMINDERS.md` and ask the intern to teach you.

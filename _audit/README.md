# `_audit/` — Audit Artifacts and the Instrumentation Stance

> Where the ecosystem looks at itself.

This directory holds:

- **Self-audit findings.** Internal consistency checks (frontmatter, cross-references, Bhishma adherence) generated by Claude Code sessions. Files: `YYYY-MM-DD_self-audit.md`.
- **External best-practice playbooks.** Deep research deliverables — tier-tagged citations, structured around principles / patterns / anti-patterns / decision tables. Files: `YYYY-MM-DD_<topic>-playbook.md`.
- **Gap analyses / decision sheets.** Synthesis of internal vs. external — what to do, ranked impact-to-effort. Files: `YYYY-MM-DD_gap-analysis.md`.

These are deliverables to Kartavya, not to any other agent. Sahadeva may read them (read-scope is the whole repo); no other agent should write to this directory.

## The instrumentation stance — what's novel here is untested

Kartavya's ecosystem uses a **four-layer oversight stack**: Tier-0 worker agents (Arjuna / Hanuman / Nakula / Narada / Yudhishthira / Vidura), Tier-1 meta-observer (Sanjaya), Tier-2 conductor (Vyasa), and Tier-Audit external auditor (Sahadeva).

**The four-layer pattern as a whole is not directly studied in the multi-agent literature** (per `2026-05-11_multi-agent-playbook.md` §8, T1+T2 sources). What exists:

- Recursive self-critique has limited replication ([Wen et al. 2025, T2] — single-source for the strongest claim).
- Weak-to-strong monitoring works at AUC ~0.85 with hybrid scaffolding ([Kale et al. 2025, T2]).
- Monitor-AUC caps near 0.87 for a single watcher ([SHADE-Arena, T1]).
- Auditor rotation is well-established in accounting (Sarbanes-Oxley) but transfer to AI is by analogy only.
- The closest empirical pattern (multi-agent debate) typically uses 3 agents × 2 rounds, not four-tier oversight.

**Implication.** This ecosystem is far enough out on the frontier that we may be among the first to produce a useful post-mortem of how four-layer AI oversight actually behaves over time. The honesty about this is itself the instrumentation.

### The open questions worth tracking

1. **Independence between tiers.** What fraction of Sanjaya's flagged patterns does Vyasa endorse independently? What fraction does Sahadeva flag that the other two missed? If the answer trends toward 100% agreement, the tiers are not adding independent signal — they're compounding the same biases.

2. **Calibration drift across model updates.** When Claude (or whichever model) gets a new training run, do Sanjaya's "concerning" thresholds drift? No empirical guidance exists; the floor is "recalibrate at least once per model update".

3. **Whether `skill.md`-style procedural memory drifts.** No published study of how the quality of a procedural document decays over time as worker behaviour evolves. The bootstrap-threshold-then-adapt rule (10 days / 20 runs; 18 days / 40 runs) is principled but not validated.

4. **Approval-fatigue thresholds in practice.** Literature gives ~10 reviews/session as the honest-review ceiling. R18 (introduced 2026-05-11) routes by risk tier to keep volume below that — but the actual fatigue threshold for this ecosystem is empirical, not a priori.

5. **Inter-agent prompt-injection.** Tier-0 worker output is read by Tier-1 observer; the worker's outputs are an attack surface for the observer. Literature exists for two-agent systems; not for four-tier.

### How we instrument

- **Quarterly tier-independence review.** Kartavya reads three randomly-selected anomalies from each tier and checks whether the framings are genuinely different. Cadence: first calendar week of each quarter.
- **MAST taxonomy in journal entries** (introduced 2026-05-11). Stable diagnostic vocabulary aligned with the largest published multi-agent failure taxonomy [Cemri et al. NeurIPS 2025, T1].
- **Adversarial test set for Sahadeva** (planned, Phase 2): 10–20 implanted-behaviour cases per quarter that measure Sahadeva's detection rate directly.
- **Structured traces** (planned, Phase 3): machine-readable spans alongside the narrative journals so cross-tier comparison can be automated.

### The discipline

When a recommendation in this ecosystem says "best practice", it must cite either:

- A tier-tagged source from a research playbook in this directory, or
- An internal pattern observed across multiple Sanjaya journals and validated by Kartavya.

Hand-waving best-practice claims ("we should add X because that's how it's done") without provenance is treated as scope creep and refused at the proposal stage.
